# tag: docs — Dokumentation (House-Rule, querschnitt)

> **Wer trägt diesen Tag:** primär die Docs-Rolle. Die Doku-**Pflicht** gilt aber teamweit (House-Rule),
> dieser Tag bündelt die geteilte Mechanik.

## Pflichten

- **Immer Docs schreiben.** Übernimmt jemand ein un-dokumentiertes System + scannt es → die Erkenntnisse
  fließen in die entstehende Doku. Audit-/Compliance-relevante Dinge werden dokumentiert.
- **Doc-Drift gegen Code melden:** wenn Code von der Doku abweicht, ist das ein Finding (CLAUDE.md-aktuell-halten).
- **Reports nach Schema ablegen** (eigener Takt), nicht ad-hoc-Dateien streuen.
- **README englisch** für veröffentlichte Engine-Teile; Konzept + Warum/Wie + Credit + Timeline.
